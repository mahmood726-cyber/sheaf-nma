## 1. Problem and goal

Phase 1 shipped the cellular-sheaf math (coboundary `F`, Laplacian `L = FᵀF`, per-edge residuals) plus an E156 micro-paper (#153, DRAFT). The current evidence base is **one synthetic 12-contrast network with planted A–D inconsistency at seed=42**. The "smoking cessation network with 24 contrasts" framing in the E156 abstract is nominal — the underlying data is synthetic.

Phase 2 goal: take SheafNMA from "math + one toy demo" to a methodology paper with empirical evidence on real published NMAs and a simulation power study against canonical comparators.

**Non-goals (deferred to v0.3+):**
- Multi-arm covariance correction (acknowledged limitation in Phase 1 abstract).
- Higher-dimensional stalks (Phase 1 spec §3.4 noted this for Phase 2; deferred again — single-D stalks are enough for the headline claim and keep the comparator math tractable).
- Interactive dashboard rework. The existing `sheaf_nma.html` ships unchanged.
- Bayesian sheaf priors.

---

## 2. Headline claim (the empirical result the paper turns on)

After running the validation pipeline on the netmeta built-in NMA corpus, the paper will report a headline of this shape:

> "Across N=15 published Cochrane / textbook NMAs spanning diabetes, smoking cessation, thrombolysis, antimanic, acupuncture, and other indications, sheaf-residual analysis flagged at least one localized-inconsistency edge in **K of N reviews** where the global design-by-treatment χ² failed to reject at α=0.05. Per-edge residual concentration agreed with node-splitting flags on **A%** of edges flagged by either method, and the simulation power study showed sheaf-residual-threshold sensitivity of **S%** and specificity of **1 − E%** for detecting single-edge inconsistency at magnitude τ_inc = 0.5 across (n_treatments × n_studies_per_edge) grid cells."

The concrete numbers come from the run. The claim shape is: *sheaf finds localized inconsistency that the global χ² misses, and the simulation backs the discovery rate.*

  test_r_bridge.py          # netmeta loader + node-splitting on Senn 2010 diabetes; skipped when Rscript not on PATH

## 5. Comparator suite

Three comparators, all per-edge or per-network:

### 5.1 Design-by-treatment interaction χ² (Higgins 2012)

Pure-Python implementation in `comparators.py`, following Higgins et al. (2012) *Research Synthesis Methods* §3. Each "design" is a unique treatment-set pattern. The test fits the full inconsistency model (treatment-by-design interaction) versus the consistency model and compares fits via a Wald χ² with `df = (number of independent design × treatment interactions) = number of designs − number of treatments + 1` (per Higgins §3.2). Returns a single p-value per NMA. **No edge-level localization** — this is the "global test" SheafNMA is meant to localize.

### 5.2 Bucher loop-closure test

Pure-Python. For each independent loop in the network, compute direct-vs-indirect difference and its standard error. Returns one z-stat + p-value per loop. Per-loop, not per-edge.

### 5.3 Node-splitting (Dias 2010, via R `netmeta::netsplit`)

R shellout via `sheafnma/r_bridge.py`. Returns per-comparison direct vs indirect estimates + difference p-value. **Per-edge localization** — direct comparator to SheafNMA's per-edge residual.

Tests for §5.1 and §5.2 hand-verify against ≥3 textbook networks (Senn 2010 diabetes is a standard worked example). Test for §5.3 runs against Senn 2010 diabetes and asserts the netmeta-reported p-values match the docstring of `netmeta::netsplit`.

## 6. Simulation power study

`analysis/power_simulation.py`:

- **Grid:** `n_treatments ∈ {4, 6, 8}` × `n_studies_per_edge ∈ {3, 5, 10}` × `τ_inc ∈ {0, 0.2, 0.5, 1.0}` = 36 cells
- **Replications:** 1000 per cell
- **Inconsistency planting:** one randomly-chosen edge per network gets effect = direct + τ_inc; all other edges remain consistent. Multi-study edges are inverse-variance pooled.
- **Outcome per replication, per method:**
  - SheafNMA: flagged-edge set = edges with |r_e| > `threshold_τ`. `threshold_τ` is selected on a held-out training grid cell (`n_treatments=6, n_studies=5, τ_inc=0.5`) by maximizing Youden's J = sensitivity + specificity − 1 over candidate thresholds, then frozen for evaluation on the remaining 35 cells. This avoids the Youden-on-null-signal trap (Youden is undefined when there's no positive class).
  - Design-by-treatment χ²: did p < 0.05?
  - Bucher: any loop with p < 0.05?
  - Node-splitting: flagged-edge set = edges with split-test p < 0.05 (with Bonferroni over edges)
- **Reporting:** type-I error at τ_inc=0 (target ≤ 0.05 each cell), power at τ_inc > 0. Edge-level sensitivity / specificity for SheafNMA + node-splitting (the two per-edge methods).
- **Determinism:** seed-locked xoshiro128ss; test_reproducibility.py asserts run-to-run identity.

This is the comparator-versus-comparator analysis the methods paper turns on.

## 7. Real-corpus validation

`analysis/run_real_corpus.py`:

- For each of the 15 netmeta built-in datasets:
  - Load as standard CSV.
  - Run SheafNMA: compute GII, per-edge residuals, eigendecomposition.
  - Run design-by-treatment χ² (global p-value).
  - Run Bucher (per-loop p-values).
  - Run node-splitting (per-edge p-values via R).
- Emit `corpus_results.csv` with columns: `nma_id, n_treatments, n_edges, n_studies, GII, design_by_treatment_p, n_loops_flagged_bucher, n_edges_flagged_sheaf, n_edges_flagged_nodesplit, edges_flagged_by_both, edges_only_sheaf, edges_only_nodesplit`.
- Headline-finding code reads this CSV and emits the manuscript's claim text + agreement matrix figure.

## 8. Paper structure (sheafnma_v2.md)

Research Synthesis Methods format, ~3500 words:

1. **Abstract** (300 words): problem (global vs local inconsistency), method (cellular sheaf), corpus (15 netmeta NMAs), simulation (36-cell grid), headline.
2. **Background** (~500 words): inconsistency in NMA, design-by-treatment χ², node-splitting, the gap between global and per-edge.
3. **Methods** (~1200 words): cellular sheaf on contrast graph, coboundary `F`, Laplacian `L = FᵀF`, GII, per-edge residuals, threshold selection by Youden, link to standard NMA inconsistency framework.
4. **Simulation study** (~600 words): grid, planting, type-I error, power, edge-level sensitivity / specificity. ROC figure.
5. **Application to real NMAs** (~500 words): 15 netmeta datasets, headline counts, agreement matrix vs node-splitting.
6. **Discussion** (~400 words): when sheaf wins, when node-splitting wins, multi-arm limitation (deferred), recommendations.
7. **References**: ≤30 entries; cite Higgins 2012 (DBT-χ²), Dias 2010 (node-splitting), Hansen-Ghrist 2019 (cellular sheaf Laplacian), Salanti 2008 / 2014.

Companion `methods_note.md` is the existing E156 #153 abstract, **rewritten** to drop the "smoking cessation network" framing (which was synthetic in v0.1) and replace with "netmeta::dias smoking cessation NMA" (the real Dias 2010 4-treatment, 24-contrast network — same domain, but real).

## 9. Tests (TDD, per AGENTS.md)

Every function in `sheafnma/core.py` and `sheafnma/comparators.py` gets tests written **before** the function is moved. Test layers:

| Layer | What it asserts |
|---|---|
| `test_core.py` | `build_coboundary(network)` returns the right shape, signs, weights. `sheaf_laplacian(F)` is PSD. `gii` returns 0 on a consistent network and >0 on an inconsistent one. `edge_residuals` reproduces hand-computed values on a 3-edge triangle. |
| `test_comparators.py` | `design_by_treatment_chi2` matches a hand-computed value for a 4-treatment 3-design example. `bucher_loop_closure` matches the Bucher 1997 worked example. |
| `test_io.py` | Standard CSV round-trips. Missing column → `KeyError` with column name in message (per the substitution-on-missing-required lesson). Empty CSV → explicit error, not silent. |
| `test_r_bridge.py` | Skipped when `Rscript` not on PATH (CI-safe). When R is available: load Senn diabetes via netmeta; run `netsplit`; assert the p-value structure shape. No exact-value assert (netmeta updates can shift digits — use range assert ≤ 1.0). |
| `test_reproducibility.py` | Two consecutive `power_simulation.py --seed 42 --cell n_treat=4,n_studies=3,tau=0.2 --reps 10` runs produce byte-identical results. |

Numerical tolerance: deterministic estimator tests use `atol=1e-9`; Monte Carlo / power tests use `atol=0.05` per the rules-pack convention.

## 10. Out of scope

- **Multi-arm covariance** — single-D stalks only. The methods paper acknowledges this as a limitation and points to it as the v0.3 follow-up.
- **CSV format extensions** — `design` column is parsed but optional; not required for v0.2.
- **Higher-dimensional stalks** (Phase 1 spec §3.4) — deferred again.
- **OTS / Bitcoin anchor** — known broken on Python 3.13 + Windows (per `lessons.md` 2026-04-29). Release tag stays as-is; OTS retried in a separate session once the toolchain issue is fixed.
- **Sentinel BLOCK clean-up** — current `STUCK_FAILURES.jsonl` and `sentinel-findings.md` are inherited from Phase 1; v0.2 commits address only NEW Sentinel hits. Pre-existing BLOCKs that re-fire on the v0.2 push get triaged but not necessarily cleared.
- **Submodule of e156-ecosystem-starter** — not added until v0.2 ships.

## 11. Risks

| Risk | Mitigation |
|---|---|
| netmeta dataset count is <15 in current package version | Run the loader script up-front to confirm; fall back to padding with `gemtc` or `multinma` datasets. The 15-dataset claim is verified before the spec ships. |
| R `netsplit` is slow on large networks (>20 treatments) | The netmeta corpus tops out around 12 treatments; not a blocker. If a future corpus is larger, batch the calls. |
| Per-edge sheaf threshold by Youden may overfit the simulation grid | Threshold selected on a single held-out cell (`n_treatments=6, n_studies=5, τ_inc=0.5`) and frozen before evaluating on the remaining 35 cells. Both numbers reported. Spec'd in `power_simulation.py`. |
| Phase 1 abstract's "smoking cessation network with 24 contrasts" sounds real — replacing it requires correcting the E156 #153 SUBMITTED state | Per `e156.md` workbook protection, E156 #153's `SUBMITTED` is `[ ]` (not frozen — DRAFT). Allowed to update the CURRENT BODY. The methods_note.md rewrite reflects this correction. |
